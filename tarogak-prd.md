# 🔮 타로각 MVP - Product Requirements Document (PRD)

**문서 버전:** v1.0  
**작성일:** 2025년 12월 23일

---

## 1. Executive Summary

**타로각**은 고민 입력하면 AI가 타로 카드 뽑아서 맞춤 해석해주는 서비스다.

```
Before: 타로샵 예약 + 2만원 + 오글거림
After: 60초 + 무료 + 혼자 편하게
```

**MVP 목표:**
- 런칭 후 30일 내 MAU 5,000명
- PMF 기준: D7 Retention 30%+, 만족도 4.0+

---

## 2. Problem Statement

### Problem 1: 결정 장애
```
현상: 사소한 고민에도 결정 못함
원인: 확신 없음, 누군가 "이렇게 해" 말해주길 바람
영향: 고민만 하다 시간 낭비, 스트레스
```

### Problem 2: 타로 접근성
```
현상: 타로 보고 싶은데 안 감
원인: 비쌈(2~3만원), 예약 귀찮음, 대면 오글거림
영향: 그냥 포기하거나 혼자 끙끙
```

**→ 해결책: 익명으로, 무료로, 60초 만에 AI 타로 리딩**

---

## 3. 타겟 유저 페르소나

### 페르소나 1: 결정장애 대학생 '김서연'
- **정보:** 23세, 대학생, MBTI F형
- **특징:** 고민 많음. 점술/심리테스트 좋아함
- **Pain Point:** "썸남이 날 좋아하는 건지 모르겠어", "휴학할까 3개월째 고민"
- **제공 가치:** 새로운 관점 제시 + 결정의 계기 제공

### 페르소나 2: 번아웃 직장인 '박준혁'
- **정보:** 28세, 개발자
- **특징:** 이직 고민 중. 점술에 관심 없었는데 뭐라도 해보고 싶음
- **Pain Point:** "이직해야 하나", "타로 보러 가기엔 오글거리고 비쌈"
- **제공 가치:** 익명으로 부담 없이, 가벼운 재미 + 의외의 통찰

## 5. User Stories & Acceptance Criteria

### Epic 1: 타로 리딩 받기 (Core Flow)

#### User Story 1.1: 고민 입력 후 카드 뽑기

```
As a 사용자
I want to 고민 카테고리와 내용을 입력하면 타로 카드가 뽑히길
So that 내 상황에 맞는 해석을 받을 수 있다
```

**Acceptance Criteria:**

✅ Given 사용자가 홈 화면에 있을 때  
   When "카드 뽑기" 버튼 클릭  
   Then 고민 입력 화면으로 이동

✅ Given 고민 카테고리(연애/취업/인간관계/재정/기타) 선택 후  
   When 구체적 고민 입력(10자 이상) 후 "뽑기" 클릭  
   Then 카드 뽑기 애니메이션 시작 (3초)

✅ Given 카드 뽑기 완료 시  
   When 결과 화면 표시  
   Then 카드 이미지 + 카드명 + 맞춤 해석(300~500자) 표시

---

#### User Story 1.2: 리딩 결과 확인

```
As a 사용자
I want to 뽑은 카드의 의미와 내 상황에 맞는 해석을 보고 싶다
So that 고민에 대한 새로운 관점을 얻을 수 있다
```

**Acceptance Criteria:**

✅ Given 결과 화면에서  
   When 카드 이미지 탭  
   Then 카드 기본 의미 팝업 표시 (정방향/역방향)

✅ Given AI 해석 영역에서  
   When 해석 텍스트 표시  
   Then 내 고민 키워드가 해석에 반영되어 있음

✅ Given 결과 화면 하단에  
   When "다시 뽑기" 버튼 표시  
   Then 클릭 시 새로운 카드 뽑기 (이전과 다른 카드)

---

#### User Story 1.3: 결과 저장 & 공유

```
As a 사용자
I want to 마음에 드는 리딩을 저장하고 공유하고 싶다
So that 나중에 다시 보거나 친구에게 보여줄 수 있다
```

**Acceptance Criteria:**

✅ Given 결과 화면에서  
   When "저장" 버튼 클릭  
   Then 내 리딩 보관함에 저장 완료 토스트

✅ Given 결과 화면에서  
   When "공유" 버튼 클릭  
   Then 카카오톡/인스타 스토리 공유 옵션 표시

✅ Given 공유 이미지 생성 시  
   When 공유 완료  
   Then 카드 이미지 + 핵심 해석 1줄 + 앱 링크 포함

## 6. Technical Architecture

### Tech Stack
```
Frontend: Next.js 15
Backend + DB + Auth: Supabase
AI: Gemini API (무료 티어)
배포: Vercel (Next.js) + Supabase (자체 호스팅)
```

### Core Data
```
타로 카드 DB: 78장 (메이저 22장 + 마이너 56장)

각 카드별 필드:
- id
- name (카드명)
- image_url (카드 이미지)
- meaning_upright (정방향 의미)
- meaning_reversed (역방향 의미)
- keywords (키워드 배열)
```

### API 핵심
```
POST /api/reading/generate
Request: { category, concern }
Response: { card, orientation, interpretation }

응답 시간 목표: P95 < 5초
```

### Gemini API 무료 티어 제한
```
- 분당 15회 요청
- 일일 1,500회 요청
- MVP 테스트용으로 충분
```

## 7. 제약 사항 및 리스크

| 리스크 | 문제 | 대응 |
|--------|------|------|
| AI 해석 품질 | 뻔하거나 엉뚱한 해석 | 카드별 의미 RAG 구축 + 타로리더 페르소나 프롬프트 |
| 반복 사용 시 식상함 | 같은 카드 = 같은 해석 | 맥락별 다른 해석 + 정/역방향 변주 |
| 윤리적 이슈 | 민감한 결정에 단정적 답변 | 면책 문구 + "~해보는 건 어떨까요?" 톤 유지 |
| API 비용 | 트래픽 증가 시 무료 한도 초과 | 일일 사용량 모니터링 + 유료 전환 대비 |

## 8. Out of Scope (MVP 제외)

```
❌ 쓰리카드/켈틱크로스 등 복잡한 스프레드
❌ 유료 결제/구독 모델
❌ 히스토리 저장/마이페이지 (MVP 이후)
❌ 소셜 로그인 (MVP 이후)
❌ 커뮤니티/댓글 기능
❌ 타로 외 다른 점술 (사주, 별자리 등)
```
